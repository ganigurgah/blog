🚀 Spring Boot Sadece REST API Sunucusu Değildir!

Embedded Tomcat'in Gücüyle Monolitik ve Full-Stack Dünyasına Yolculuk

📚 Kategori: Geliştirme / Mimari
⏱️ Okuma Süresi: ~12 dakika
👥 Hedef Kitle: Java & Web Geliştiricileri

📖 Giriş: Yaygın Yanlış Algı

Günümüz modern web ekosisteminde öyle keskin sınırlarla karşılaşıyoruz ki, yeni nesil geliştiricilerin büyük bir kısmı backend ve frontend mimarilerinin tamamen izole olması gerektiğine inanıyor. Bu algının merkezinde genellikle şu basitleştirme yatar:

❌ Yaygın Yanlış Kanı: "Spring Boot sadece JSON veri dönen bir REST API sunucusudur; arayüz ise harici bir sunucuda (Vercel, Netlify, AWS S3 vb.) barındırılan bir React/Vue uygulaması olmak zorundadır."

Bu varsayım tamamen yanlıştır! Spring Boot, kalbinde yerleşik (embedded) bir Apache Tomcat sunucusu taşır. Bu entegrasyon, onun sadece veri yöneten hafif bir servis değil, aynı zamanda uygulamanın tüm katmanlarını (Full-Stack) tek bir çatı altında sırtlayabilen, mikroservis karmaşasından uzak, dağıtımı kolay ve son derece performanslı monolitik yapılar kurabilen devasa bir web sunucusu olduğu anlamına gelir.

Bu makalede, Spring Boot'un gömülü gücünü kullanarak arayüz teknolojilerini nasıl doğrudan tek bir .jar dosyasında birleştirebileceğimizi 4 farklı yöntemle, mimari ve pratik kod yapılarıyla ele alacağız.

🏗️ Mimarinin Temeli: Single JAR Dağıtımı ve Avantajları

Her şeyi tek bir Spring Boot projesinde toplamanın (Single JAR Deployment) bağımsız geliştiriciler (Indie Hackers) ve kurumsal monolit projeleri için benzersiz avantajları vardır:

✅ Sıfır CORS Problemi Frontend ve backend aynı port ve domain üzerinden (localhost:8080) haberleştiği için tarayıcıların meşhur Cross-Origin Resource Sharing engelleri tamamen ortadan kalkar.
💰 Minimal Sunucu Maliyeti Ayrı bir Node.js veya statik dosya sunucusuna ihtiyaç kalmaz. Tek bir Linux sunucusu veya Docker container içinde çalışan tek bir .jar dosyası tüm web sitesini ayağa kaldırır.
🔄 Kolay CI/CD Sadece tek bir Maven veya Gradle build süreciyle hem arayüz derlenir hem de sunucu paketi hazır hale gelir.

📁 Gömülü Tomcat Dosya Sunumu Yapısı

Dizin Yolu Sunulan İçerik Türü Erişim Tipi
/src/main/resources/static/ Derlenmiş React static (JS/CSS), resimler, düz HTML Doğrudan sunucu üzerinden statik dosya erişimi
/src/main/resources/templates/ Thymeleaf şablonları, JSP dosyaları Sunucu tarafında dinamik render (SSR)

1Spring Boot + Thymeleaf / JSP

📄 Geleneksel Sunucu Tarafı Şablonlama (SSR)

Sunucu tarafında şablonlama (Server-Side Rendering), verinin istemciye daha gitmeden HTML içine gömülmesini sağlar. Güvenlik ve SEO avantajı yüksektir.

🔌 Thymeleaf Entegrasyonu

Modern Spring ekosisteminde JSP yerine varsayılan şablon motoru olarak Thymeleaf önerilir. Tamamen geçerli HTML5 yapısını korur.

Maven Bağımlılığı (pom.xml)

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-thymeleaf</artifactId>
</dependency>

Controller Katmanı

@Controller
public class HomeController {
@GetMapping("/selam")
public String selamla(Model model) {
model.addAttribute("mesaj", "Spring Boot ve Thymeleaf Birlikteliği!");
return "home"; // src/main/resources/templates/home.html dosyasını arar
}
}

Şablon Dosyası (src/main/resources/templates/home.html)

<html xmlns:th="http://www.thymeleaf.org">
<body>
<h1 th:text="${mesaj}">Varsayılan Metin</h1>
</body>
</html>
⚠️ JSP/War Notu: Eğer eski usul JSP kullanmak isterseniz, Spring Boot projenizi standart bir .jar yerine .war olarak paketlemeli ve tomcat-embed-jasper bağımlılığını eklemelisiniz. Ancak modern yapılar için Thymeleaf çok daha esnektir.

2Spring Boot + Vaadin ile Tam Entegre React Uygulamaları

Vaadin (v22+), Java geliştiricilerin tek bir satır JavaScript veya CSS yazmadan, tamamen Java nesneleriyle modern ve reaktif single-page web uygulamaları (SPA) geliştirmesini sağlar.

Arka planda Vaadin, Java kodlarınızı optimize edilmiş React bileşenlerine dönüştürür ve istemci ile sunucu arasındaki iletişimi WebSocket (Atmosphere framework) veya optimize edilmiş XHR ile otomatik yönetir.

Maven Bağımlılığı (pom.xml)

<dependency>
    <groupId>com.vaadin</groupId>
    <artifactId>vaadin-spring-boot-starter</artifactId>
</dependency>

Java ile Arayüz Geliştirme

@Route("") // Uygulamanın kök dizininde (/) çalışır
public class MainView extends VerticalLayout {
public MainView() {
TextField adInput = new TextField("Adınız:");
Button buton = new Button("Gönder");

        buton.addClickListener(click -> {
            Notification.show("Merhaba, " + adInput.getValue() + 
                            "! Bu bir React WebApp'tir.");
        });
        add(adInput, buton);
    }
}

Vaadin, projeyi derlediğinizde otomatik olarak bir Node.js entegrasyonu kurar, gerekli React bileşenlerini arka planda hazırlar ve gömülü Tomcat üzerinden tarayıcıya sunar. JavaScript dünyasına girmeden React gücünü kullanmak isteyenler için kusursuzdur.

3Spring Boot + JSF (PrimeFaces / BootsFaces) Mimarisi

Özellikle kurumsal ve veri yoğunluklu (data-centric) legacy sistemlerin modernizasyonunda JSF (JavaServer Faces), zengin bileşen kütüphaneleriyle hala çok güçlüdür. Spring Boot çatısı altında PrimeFaces ve BootsFaces bileşenlerini bir arada kullanarak modern tasarımlar elde etmek mümkündür.

Spring konfigürasyonunda JSF'in FacesServlet bileşenini gömülü Tomcat'e elle kaydetmemiz gerekir.

Servlet Kayıt Konfigürasyonu

@Configuration
public class JsfConfig {
@Bean
public ServletRegistrationBean<FacesServlet> facesServletRegistration() {
ServletRegistrationBean<FacesServlet> registration =
new ServletRegistrationBean<>(new FacesServlet(), "*.xhtml");
registration.setLoadOnStartup(1);
return registration;
}
}

PrimeFaces ve BootsFaces Karışımı XHTML (src/main/webapp/index.xhtml)

<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:h="http://xmlns.jcp.org/jsf/html"
      xmlns:p="http://primefaces.org/ui"
      xmlns:b="http://bootsfaces.net/ui">
<h:head>
<title>JSF ve Spring Boot</title>
</h:head>
<h:body>
<b:container> <!-- BootsFaces Grid Sistemi -->
<b:jumbotron>
<h2>Enterprise Çözüm: JSF & Spring Boot</h2>
</b:jumbotron>
<h:form>
<!-- PrimeFaces Gelişmiş Bileşeni -->
<p:colorPicker value="#{jsfBean.renk}" />
<p:commandButton value="Kaydet" action="#{jsfBean.kaydet}" />
</h:form>
</b:container>
</h:body>
</html>

Bu mimari sayesinde, kurumsal arka plan servisleriniz Spring bağımlılık enjeksiyonu (DI) ile yönetilirken, ön yüzü JSF'in binlerce hazır kurumsal bileşeniyle ayağa kaldırabilirsiniz.

4Saf React Uygulamasını Gömülü Tomcat Klasörüne Çakma

🎯 The Indie Hacker Way

Bağımsız bir frontend ekibiniz varsa veya harici bir React repoyu tek bir JAR içinde sunmak istiyorsanız, en agresif ve performanslı yöntem budur. Süreç tamamen otomatize edilebilir.

📋 Yol Haritası ve Otomasyon

  1. React projenizde terminale gidip npm run build komutunu çalıştırırsınız (Vite kullanıyorsanız da aynı).
  2. Derleme sonucu oluşan dist/ veya build/ klasörü altındaki tüm dosyaları (index.html, assets/ klasörü vb.) kopyalarsınız.
  3. Bu dosyaları Spring Boot projenizin src/main/resources/static/ dizinine yapıştırırsınız.

Maven Plugin ile Otomasyon

Bunu her seferinde elle yapmak yerine Maven derleme sürecine bağlayarak frontend-maven-plugin ile otomatize edebilirsiniz: 

<plugin>
    <groupId>com.github.eirslett</groupId>
    <artifactId>frontend-maven-plugin</artifactId>
    <version>1.12.1</version>
    <!-- Konfigürasyon ile npm install ve npm run build adımları 
         jar paketlenirken otomatik çalışır -->
</plugin>

🚨 Kritik İpucu: React Router için Sunucu Yönlendirmesi

React tek sayfalık bir uygulamadır (SPA). Eğer kullanıcı direkt tarayıcıya domain.com/dashboard yazarsa, Tomcat bu dizinde fiziksel bir dosya arayacak ve bulamadığı için 404 hatası dönecektir.

Bunu engellemek için tüm bilinmeyen istekleri React'in index.html dosyasına yönlendiren bir Controller yazmalısınız: 

@Controller
public class SpaController {
@RequestMapping("/{path:[^\.]*}")
public String redirectAll() {
return "forward:/"; // İsteği istemci tarafındaki yönlendiriciye (React Router) paslar
}
}

🎯 Sonuç: Ne Zaman Hangi Yöntem?

Gördüğünüz gibi Spring Boot, gömülü Apache Tomcat sunucusu sayesinde tek başına devasa bir ekosistemi sırtlayabilir. Doğru mimariyi seçmek projenizin ihtiyaçlarına bağlıdır:

📊 Yöntemler Karşılaştırması

Yöntem Kullanım Alanı Zorluk Derecesi Performans
Thymeleaf / JSP Hızlı, temiz ve SEO uyumlu içerik tabanlı projeler Kolay ⭐ Mükemmel 🚀
Vaadin JavaScript yazmadan saf Java ile modern React SPA Orta ⭐⭐ Çok İyi ⭐⭐⭐⭐
JSF + PrimeFaces Veri yoğunluklu ve legacy kurumsal projeleri modernize etmek Zor ⭐⭐⭐ İyi ⭐⭐⭐
Static React Injection Maksimum esneklik, sıfır CORS hatası, ayrışmış frontend Orta ⭐⭐ Mükemmel 🚀

✅ Şu Durumlar İçin Öneriler

📰 Hızlı, Temiz ve SEO Uyumlu İçerik Tabanlı Projeler → Thymeleaf
⚙️ JavaScript Yazmadan Saf Java ile Modern SPA → Vaadin
🏢 Veri Yoğunluklu Legacy Kurumsal Projeler → JSF + PrimeFaces/BootsFaces
🎨 Maksimum Esneklik ve CORS Problemi Olmayan Ayrışmış Frontend → Static Folder React Injection

Bir sonraki projenizde sunucu sunucu gezmeden önce, elinizdeki gömülü Tomcat'in sınırlarını zorlamayı unutmayın! 🎯
